大学三年，我是怎么一步步走上算法这条路的

发表于 2025-11-24

大学经历 · 一点不端着的真心话

如果你现在还在想「我是不是已经晚了」「我大学到底在干嘛」，那这篇大概就是写给你的。

大一（2022.09 — 2023.06）：

从「什么是指针」到敢说一句「我好像有点喜欢写代码」

大一对我来说，其实并不是什么「天赋异禀，一出场就开挂」的故事，反而是很普通、甚至有点迷茫的开始。

刚进大学那会儿，我和大多数同学一样，按部就班地上《C语言》《C++》课，刷一点基础的算法题，while、for、数组、指针……当时对“工程”“项目”这种词完全没概念，只是觉得：把一堆符号敲进电脑，能换来一行对的输出，这件事挺神奇的。

大一下学期是一个转折点。
我进了一个本科生实验室，第一次接触到「不为考试写代码」这件事——学长丢给我一个小任务，然后说一句：

“你先自己试试，不会就问我或者问 AI。”

那段时间，我一口气补完了 Python 和 Vue3 的基础。
我没有去从头到尾看完一本书，而是用 AI 反推自己的知识缺口：

  • 想实现一个功能，就先问 AI 需要哪些语法；
  • 看不懂报错，就把报错直接丢给 AI，一条条照着改；
  • 有些概念搞不清，就让它用「我能听懂的方式」再解释一遍。

我印象特别深的是，学 Python 基础语法，大概三四个小时就过了一轮，然后晚上又用两三个小时，把它用在一个小脚本里——那一刻我第一次意识到：

原来「会不会」这件事，很多时候差的不只是时间，而是方法。

大一下我还跟着实验室的学长一起报名了数学建模和大学生计算机设计大赛。
第一次参赛，过程远比结果重要：

  • 熬夜赶 PPT、改代码、对着数据一遍遍调参数；
  • 第一次知道「原来论文里的模型真的可以自己实现出来」；
  • 也第一次意识到，比赛其实是帮你快速补全「能力拼图」的地方——建模、写作、表达、协调、落地，这些以后找实习、保研都绕不过去。

说实话，大一这一年，我还没完全想清楚未来要干嘛，但我做对了一件事：多去试。

大二（2023.09 — 2024.06）：

方向感出现：一边卷课程，一边偷偷搞网站和 AI

1. 学业：从「为成绩而学」到「为选择而学」

大二上的课开始「变硬核」：数据结构、离散数学……这些课一开始我也觉得枯燥，但慢慢明白：

这些东西短期看不到成就感，但长期是底层“操作系统”。

这一年，学校创办了国家特色示范性软件学院，本硕贯通班的机会摆在眼前：
笔试 + 机试 + 面试 三轮过后，我拿到了入班资格——
这意味着：只要成绩达标，将来保研不再是一个遥不可及的词。

也正是这一点，给了我一个很大的心态变化：

  • 从「疯狂卷绩点」变成「绩点稳住，拿时间去做更有增长的事」。
    这一点对我后面所有决定都非常关键。

2. 折腾网站：第一次花自己钱租云服务器

大二我特别迷恋一件事：自己搭网站。
不是课程作业，而是真正跑在公网、别人能访问的那种。

我花钱租了云服务器，从零配环境、买域名、备案、部署。
常用的技术栈慢慢稳定下来：

  • 前端：Vue3
  • 后端：Django
  • 部署：Docker 一把梭

这些项目里，很多页面现在回头看都很青涩，但它们让我学会了几件更重要的事：

  • 需求不会一开始就清晰，边做边改是常态；
  • 文档永远看不完，但**“先抄一个能跑的，再改成自己的”**是个很好用的方法；
  • 线上环境跟本地永远不一样，踩坑能力本身也是一种能力。

3. AI 爆火的那一年：我被狠狠种草了

2023 年，ChatGPT 火得一塌糊涂。
我第一次试用 GPT 的时候，几乎是震惊的：

这是我从小到大见过最像“会思考”的软件。

于是我开始系统性地学机器学习和深度学习。

  • 跟着沐神的《动手学深度学习》一点点推公式、敲代码；
  • 用 PyTorch 重写课上的一些小模型；
  • 为了给 PyTorch 配好一个干净的 conda 环境，硬是折腾了好几天（现在已经是「看到 CUDA 报错都不会慌」的程度了）。

在此基础上，我顺着好奇心往下走，开始接触 NLP：

  • 在实验室的 3090 上部署 ChatGLM-8B；
  • 自己写 API，做了一个小型的对话系统；
  • 还尝试给模型做过简单的微调，让它适配我们自己的数据。

那段时间，我第一次有了非常清晰的念头：

“我以后想做的，应该就是这一类工作 —— 和模型、数据、算法打交道。”

大三（2024.09 — 2025.06）：

聚焦一条赛道：科研、工程、比赛、实习串到了一起

如果说前两年是「广撒网」，大三就是：我开始认真给自己定方向了：深度学习 / 算法工程。

1. 走进研究生实验室：从「会写代码」到「会读论文」

我从原来的本科生实验室，转到了一个研究生类型的实验室。
组里全是研究生和博士，组会的第一天，我就感受到了「信息密度差不多翻倍」：

  • 每周组会需要汇报论文；

  • 一篇 paper 不再只是「看一眼结论」，而是要搞明白：

    • 问题是什么？
    • 现有方法卡在哪？
    • 作者的点子到底新在哪？
    • 模型、损失、训练细节能不能复现？

刚开始我也看得很痛苦，英文原文+公式双重暴击，但坚持了几个月之后，真的能感觉到：

大脑好像被「调过频」了，看新东西的速度和深度都不一样了。

2. 做多模态项目：第一次在真实单位部署自己的系统

在导师的安排下，我参与了一个多模态知识图谱系统的横向项目，技术栈大致是：

  • 文本侧：BERT 做实体抽取
  • 图像侧：Yolov5 做目标检测
  • 后端：Flask API
  • 知识存储：Neo4j
  • 部署：Docker 打包、一键部署

我在其中负责的工作包括：

  • 设计整体数据流和接口；
  • 整合各模块的输出，落到知识图谱结构上；
  • 带着组里的同学分工协作，负责进度推进。

后来我们把这个系统带到北京的一家研究所，做了离线部署。
那是我第一次在「真正的单位」里，给完全不懂代码的老师演示自己做的东西，帮他们在自己的服务器上跑起来。
那一刻我很清晰地感受到：

原来代码是可以从实验室走到现实场景里的。
而我，是可以真的把东西落地的人。

同时，我也对国企、研究所的工作节奏、沟通方式有了非常直观的认识——这些都是课本不会教、但对未来选择很重要的东西。

3. Kaggle / 天池：学会「和数据好好相处」

大三我还和大四学长组队，参加了 Kaggle 和天池上的两个课题。
结果不算光鲜：没有上榜，也没有拿牌。

但这段经历给我的收获非常直接：

  • 深刻体会到：模型只是冰山一角，数据才是地基；
  • 学会了花大量精力在数据清洗、特征构造、数据划分、评估指标上；
  • 才明白为什么大佬们总说：“Garbage in, garbage out.”

这也慢慢影响了我看待「算法工程师」这个岗位的视角：

不是去追某个“最新模型名字”，而是用合适的工具解决真实的问题。

4. 第一次正式走进大厂：华为 AI 算法实习 Offer

大三下（春季）开始，我一边上课，一边开始系统地准备简历、刷岗位、投实习。

做简历的时候，我把这三年所有做过的东西摊开，冷静问自己两件事：

  1. 哪些是能真正代表我技术水平的？
  2. 哪些是我未来想往下走的方向？

于是我把方向锁定在「AI / 算法」相关岗位，开始有选择地准备面试、补基础、梳理项目。
最后，我拿到了华为的一段暑期实习（AI 算法实习工程师）的机会。

这个结果没有让我觉得「我已经很牛」，反而更像是一个提醒：

过去三年的所有选择，在这里暂时汇总成了一个阶段性反馈 ——
说明我走得还可以，但也远远没到可以停下来的时候。

写在最后：给正在读大学的你的一些不那么「鸡汤」的建议

如果你看到这里，可能你也正在经历类似的迷茫：

  • 专业是不是选对了？
  • 别人都走在“很清晰的路上”，但我好像啥都想试试？
  • 我现在开始努力，会不会已经晚了？

我想用我这三年踩出来的坑，给你几个非常真诚的小建议：

1. 大一大二，多试；大二下大三上，学着「收敛」

  • 大一可以多去试：Web、算法竞赛、嵌入式、AI……都可以。
  • 但不要一直停留在「尝一口又换一个」的状态。
  • 大二下 / 大三上，是一个很适合「择一深耕」的时间节点。

你可以问自己几个问题：

  • 哪件事，是我愿意不拿学分也会去做的？
  • 哪个方向，是我愿意为它多啃几篇英文论文的？
  • 哪个项目，是我回想起来会有一点点自豪感的？

答案基本就在那里。

2. 把「工具」用到极致，而不是用来逃避

我非常推荐你在学习中好好用 AI：

  • 不会的报错丢给 AI，让它帮你定位问题；
  • 新技术栈先让 AI 给你一个「最小可跑 demo」；
  • 不懂的概念让 AI 用不同方式解释，直到你真的听懂。

但同时要记得：

AI 是放大器，不是替身。
你不想动脑，它帮不了你太多；
你愿意思考，它会让你效率翻几倍。

3. 比赛、实验室、实习：不是「必须做满三件套」，而是问自己要什么

  • 比赛，给你的是「短期极高压下的快速成长」，和可观的履历加分；
  • 实验室，给你的是「向上看的机会」，学会怎么做研究、怎么跟导师和学长学习；
  • 实习，给你的是「向外看的机会」，知道行业到底在做什么。

你不一定三样都做，但建议至少有一两个能拿得出手的：

  • 比如：你在一个项目里，从头到尾负责了关键模块；
  • 或是：你能清晰讲清楚一篇论文、一个方法，以及它的优缺点。

4. 规划，不是画一个「完美路线图」，而是学会不断迭代

我现在回头看，会发现自己并不是一开始就「奔着算法工程师」去的。
很多时候是：

  • 先做了某件事，
  • 在过程中发现自己喜欢或者不喜欢，
  • 然后再调整方向。

如果你现在还没想清楚未来要做什么，很正常。
但至少可以从这几件事开始：

  • 养成用文档和仓库记录成长的习惯（GitHub、博客、Notion 都行）；
  • 每个学期结束的时候，给自己写一个「这一学期我做成了什么」的总结；
  • 每次做选择时，问一句：“这件事是把我推向我想要的生活，还是在消耗我？”

真的最后：如果你看到这里

如果你也在读计算机，如果你也在为「未来的路」焦虑，
你可以把你的情况简单写给我：

  • 你现在几年级？
  • 在学什么 / 想学什么？
  • 对哪条路有一点点心动？

如果你愿意，我们甚至可以一起，把你接下来一年的计划拆成几条清晰的时间线。

大学三年对我来说不是「一路顺风」，而是「边摔边改，偶尔抬头能看到一点远处的光」。
如果这篇经历能让你少踩一点坑，那就值了。
